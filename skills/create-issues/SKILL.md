# Output Formatter: Issue Creator

## Purpose
Transform code review findings into trackable issues for deferred fixes. Groups related findings, assigns labels, and provides actionable guidance.

## Issue Template

```markdown
# [SECURITY] SQL Injection in User Authentication

## Severity: 🔴 Critical

## Description
SQL injection vulnerability detected in user authentication flow. User input is directly interpolated into SQL queries without parameterization.

## Affected Files
- `src/auth.py:42`
- `src/api/users.py:105`

## Evidence

### src/auth.py:42
```python
query = f"SELECT * FROM users WHERE username = '{username}'"
cursor.execute(query)
```

### src/api/users.py:105
```python
User.objects.raw(f"SELECT * FROM users WHERE id = {user_id}")
```

## Impact
Attackers can bypass authentication, extract sensitive data, or execute arbitrary SQL commands.

## Fix Guidance

Use parameterized queries:

```python
# auth.py
query = "SELECT * FROM users WHERE username = %s"
cursor.execute(query, (username,))

# users.py
User.objects.filter(id=user_id)
```

## Related PR
#<pr_number>

```

## Grouping Strategy

### Group by Category
```python
def group_findings(findings):
    groups = {}
    for finding in findings:
        key = (finding.category, finding.subcategory)
        if key not in groups:
            groups[key] = []
        groups[key].append(finding)
    return groups

# Result: {('security', 'sql-injection'): [finding1, finding2], ...}
```

### One Issue Per Group
```python
for (category, subcategory), findings in groups.items():
    issue_title = f"[{category.upper()}] {subcategory.replace('-', ' ').title()}"
    issue_body = format_issue_body(findings)
    create_issue(issue_title, issue_body, labels=[category, severity, subcategory])
```

## Issue Prioritization

### Create Issue When:
- Finding is **non-blocking** (doesn't prevent merge)
- Fix is **complex** (needs design discussion)
- Finding is **low/medium severity**
- Multiple instances need **coordinated fix**

### Don't Create Issue When:
- Finding is **critical/high** and **blocking** (use inline comments + PR block)
- Fix is **simple** and **automatable** (use fix-pr instead)
- Finding is **duplicate** of existing issue

## GitHub API Integration

```python
import requests

def create_github_issue(repo, title, body, labels):
    url = f"https://api.github.com/repos/{repo}/issues"
    headers = {
        "Authorization": f"Bearer {GITHUB_TOKEN}",
        "Accept": "application/vnd.github+json"
    }
    data = {
        "title": title,
        "body": body,
        "labels": labels
    }
    response = requests.post(url, json=data, headers=headers)
    return response.json()
```
