# Output Formatter: Fix PR Generator

## Purpose
Create an automated pull request that fixes code review findings. Applies suggested fixes, commits changes, and opens PR with summary.

## Generation Flow

```
1. Create fix branch (fix/auto-review-<timestamp>)
2. Apply fixes from findings
3. Commit changes with descriptive message
4. Push branch
5. Open PR with findings summary
6. Link to original PR
```

## Implementation

### Step 1: Create Branch
```bash
git checkout -b fix/auto-review-$(date +%s) origin/main
```

### Step 2: Apply Fixes
```python
def apply_fixes(findings):
    for finding in findings:
        if not finding.is_automatable:
            continue
        
        file_path = finding.file
        content = read_file(file_path)
        
        # Apply fix (line-based replacement)
        lines = content.splitlines()
        start_line = finding.line - 1
        end_line = finding.end_line - 1 if finding.end_line else start_line
        
        # Replace lines
        fixed_lines = lines[:start_line] + \
                      finding.fixed_code.splitlines() + \
                      lines[end_line + 1:]
        
        write_file(file_path, '\n'.join(fixed_lines))
```

### Step 3: Commit
```bash
git add -A
git commit -m "fix: auto-apply code review findings

- Fix SQL injection in auth.py:42
- Fix XSS in api.py:105
- Add missing CSRF protection in views.py:78

Auto-generated by code reviewer
Original PR: #123"
```

### Step 4: PR Description Template
```markdown
# 🤖 Automated Code Review Fixes

This PR applies automated fixes for findings from #<original_pr_number>.

## Fixes Applied (3)

### 🔴 Critical: SQL Injection (auth.py:42)
**Before:**
```python
query = f"SELECT * FROM users WHERE id = {user_id}"
```

**After:**
```python
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))
```

### 🟠 High: XSS (api.py:105)
**Before:**
```javascript
element.innerHTML = userInput;
```

**After:**
```javascript
element.textContent = userInput;
```

### 🟡 Medium: CSRF Protection (views.py:78)
**Before:**
```python
@app.route('/delete', methods=['POST'])
def delete_user():
```

**After:**
```python
@app.route('/delete', methods=['POST'])
@csrf_protect
def delete_user():
```

## Manual Review Needed

The following findings require manual review:
- **High: Race Condition** (payment.py:220) - Complex fix, needs human review

---

**Automated by Code Reviewer** | Original PR: #<pr_number>
```

## Automatable Fixes

### High Confidence (auto-apply)
- Add missing CSRF decorator
- Fix SQL parameterization (simple cases)
- Add missing null checks (simple)
- Fix XSS in innerHTML
- Add missing timeouts
- Fix hardcoded secrets → env vars (with placeholder)

### Low Confidence (manual review)
- Complex logic changes
- Multi-file refactoring
- Race condition fixes
- Algorithm changes

## Safety Checks
1. **Test suite must pass** before opening PR
2. **Linter must pass** after fixes
3. **No breaking changes** to public APIs
4. **Require human approval** for merge
